# Field data для урока «Рестил-джем 25–40bb» (BB/SB vs CO/BTN open)

Датасеты о том, как ведут себя РЕАЛЬНЫЕ оппоненты разных типов: как широко
открывают «чисто» и как реагируют на рестил-джем. Питают тумблер урока
«тип опенера»: хороший рег / слабый рег / нит / агрофиш / пассивный фиш и т.д.

Источник: ClickHouse `analytics` (MCP). Дата выгрузки: 2026-07-11.

## Файлы

| `field_vs_jam.json` | Реакция опенера на рестил-джем (fold/continue) по типам × бенд | 2026-01..2026-06 (6 мес.) |
| `field_call_range.json` | РУКИ, с которыми поле продолжает против джема, по типам | **2026-04..2026-06 (3 мес. — сужено, см. ниже)** |
| `../tools/01_field_opens.sql` | SQL первого датасета | |
| `../tools/02_field_vs_jam.sql` | SQL второго датасета | |

Все `*_pct` и `open_size_mix` — ДОЛИ в [0,1] (умножай на 100 для процентов).

### Почему call_range на 3 месяцах

6-месячный джойн `int_tracker_hand_joined → stg_tracker__hand_stats`
(async job `mcp_ch_job_f4819bd514824a5d994887a92ef740a9`) не завершился за ~20 минут
и был брошен. Суженный до `2026-04-01..2026-06-01` синхронный прогон выполнился
за 113 секунд и дал 28 259 продолжений против 114 977 джем-рук героев —
объёма достаточно для всех категорий, кроме нита (см. ниже).

## Определения

### Категории (tracker_united_player_cats)

`multiIf`-приоритет: horoshiy_reg → sredniy_reg → slabiy_reg → nit → aggro_fish →
passivniy_fish → polupassivniy_fish → aggressivniy_i_neustupchiviy →
agressivniy_i_ustupchiviy → 'unknown'.

Проверено: булевы почти взаимоисключающие — из 1 497 923 игроков лишь 18 строк
с >1 категорией (решает приоритет) и 2 153 строки без единой (падают в
'unknown' вместе с несматчившимися никами). Дубликатов (network, nickname)
НЕТ — join 1:1, версионирования нет.

Маппинг: good_reg, mid_reg, weak_reg, nit, aggro_fish, passive_fish,
semipassive_fish, aggro_sticky (агрессивный и неуступчивый),
aggro_foldy (агрессивный и уступчивый), unknown.

### field_opens — «чистый опен»

Строки `int_tracker_hand_joined_for_opp_id`: position IN (BTN=0, CO=1),
`is_3_9_max=1`, `cnt_players>=5`, `is_preflop_unopened=1`,
`stack_size_bb` — СОБСТВЕННЫЙ стек опенера в бенде 25-30/30-35/35-40
(чистота, которую просил тренер: без прослойки коротышей). Дедуп не нужен:
`hand_player_id` уникален в таблице (проверено на майской партиции).

- `open_any_pct` = is_rfi / возможности
- `open_clean_pct` = is_rfi И рейз ≤ 3.0bb (poker-стандартный «мелкий» опен)
- `open_jam_pct` = is_rfi И (рейз ≥ 10bb ИЛИ ≥ 85% стека) — опен-джемы
- `open_size_mix` = {le2.2, 2.2-3, 3-10, gt10} — доли среди is_rfi (сумма = 1)
- `limp_pct` = is_preflop_limp / возможности
- `category_share` = микс поля: доля возможностей каждой категории
- `pooled_25_40` — агрегат по трём бендам; `pooled_ge25` — все стеки ≥ 25bb
  (бенд 'gt40' захвачен тем же сканом)

### field_vs_jam — реакция опенера на джем

Те же opp-строки: их чистые опены (BTN/CO, is_rfi, рейз ≤ 3bb), столкнувшиеся
с 3бетом размером `amt_preflop_3bet_facing / bb_amount` в 22..45bb — прокси
рестил-джема 25-40bb; эффективный стек `preflop_effective_stack_size_bb` 25..40.
`fold_pct` = is_preflop_fold_to_3bet; `continue_pct` = 1 − fold (колл или
ре-джем — виллан уже олл-ин, оба исхода = «деньги в банк»).

### field_call_range — руки продолжений

P4-джойн: джем-руки героев проекта из `int_tracker_hand_joined`
(спот: SB/BB, val_preflop_action_facing=4, first aggressor CO/BTN, 0 лимперов,
eff 25-40bb, could_3bet, user_id NOT NULL; джем = preflop_action='R' И
is_preflop_allin=1) → `hand_id`+месяц → `stg_tracker__hand_stats`
(is_preflop_face_3bet=1, action IN ('C','R'), позиция ≠ позиция героя)
→ `stg_tracker__players` → cats → `stg_tracker__lookup_holecards`.

КТО продолжает (проверено отдельным запросом, Apr–Jun):
- BTN 47.3% + CO 30.3% = **77.6% — сам опенер** (основной кейс урока);
- BB 18.3% — второй блайнд холодно коллирует, когда герой джемит из SB;
- прочие позиции 4.1% — редкие limp-call цепочки. Итого 28 259 продолжений.
Для вопроса урока «какие руки поле доезжает против джема» все три группы валидны.

`act_split`: C = колл, R = ре-рейз поверх (изоляция от игроков сзади / кавер).
`hands` — счётчики по 169 label'ам ('AA', 'AKs', 'AKo', ...); `hand='unknown'` —
карты не записаны (без шоудауна): таких всего ~0.2% (олл-ин почти всегда вскрывается).

Мелкие категории: **только nit < 300 продолжений (n=104)** — его карманный
рэндж шумный, для нита бери `super_groups.reg`. Супер-группы reg
(good/mid/weak/nit) = 13 040, fish (все фиши+агро-типы) = 14 419, unknown = 800.

## Sanity checks (все пройдены)

1. **good_reg опенит шире нита** ✓ — clean BTN 25-30bb: good_reg 47.3% vs nit 33.6%
   (mid 41.7%, weak 35.6% — монотонно по силе рега).
2. **Реги clean BTN open в 40-60%** ✓ для good_reg (47.3% на 25-30, 54.1% на 35-40);
   weak_reg 35.6% чуть ниже коридора — это реальность выборки, порядок категорий верный.
3. **passive_fish коллирует джем чаще нита** ✓ — continue pooled: 49.8% vs 21.8%.
4. **fold_pct всех категорий в 40-85%** ✓ — от 50.1% (passive_fish) до 81.2% (aggro_foldy).
5. Сумма size-mix бакетов = open_any точно (sz_null = 0 во всех ячейках).
6. Джойн cats без дублей; hand_player_id уникален; hand_stats без версионных дублей
   (15 147 879 строк = столько же distinct hand_player_id на майской партиции).

## Каверзы и ограничения

- **Окно call_range уже, чем у остальных** (3 мес. vs 6) — сезонного смещения
  не ожидаем, но при сшивке датасетов в UI это стоит помнить.
- **Прокси джема в field_vs_jam** — размер 3бета 22-45bb, а не флаг all-in:
  включает и редкие не-олл-ин 3беты этого размера (при eff 25-40 это фактически
  им эквивалентно). В field_call_range джем строгий (is_preflop_allin=1).
- **~30% рук героев не имеют строк оппонентов в hand_stats** (известно из прежнего
  исследования, смещения нет) — n продолжений занижен равномерно, ДОЛИ рук честные.
- **field_opens и field_vs_jam считаются по всем наблюдаемым игрокам** — включая,
  возможно, самих про игроков проекта, попавших в HH друг друга (не отфильтровать
  на уровне opp-таблицы; доля мала и на «портрет поля» влияет незначительно).
- **nit**: 104 продолжения — рэндж колла нита в by_category шумный; используй
  super_groups. Его open/fold-метрики (n≈2k) пригодны.
- val_preflop_action_facing=4 у героя не исключает лимп-фолдов до опена в 100%
  случаев (см. 4.1% продолжений с ранних позиций) — шумовой хвост, не влияет
  на выводы.
- aggro_foldy — самый фолдящий тип (81.2%) при самом широком клине-опене (50-60%):
  идеальная жертва рестила; passive_fish — противоположный полюс (не опенит,
  но доезжает каждый второй раз). Это и есть учебная ось урока.

## Как пересобрать

1. Выполнить `tools/01_*.sql` и `tools/02_*.sql` (format=csv_file) → скачать как
   `opens_raw.csv`, `vsjam_raw.csv`; `tools/03_*.sql` (окно по месту) → `call_range_raw.csv`.
2. `SCRATCH=<dir с csv> DATA=<этот каталог> python3 tools/build_datasets.py`.
