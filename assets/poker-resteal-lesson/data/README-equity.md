# Preflop all-in equity dataset (169x169)

Data consumed by the resteal-jam lesson's browser calculator.
Generated by `../tools/gen_equity169.py` (python3 + numpy only, fixed seed —
fully reproducible; the JSON files themselves have no runtime dependencies).

## Files

| File | Contents |
| --- | --- |
| `equity169.json` | `{"hands": [169 labels], "equity": [[169 x 169 floats]]}` — `equity[i][j]` = P(hand_i beats hand_j) + 0.5 * P(tie), preflop heads-up all-in |
| `rank_vs_random169.json` | `{"hands": [same 169 labels], "equity_vs_random": [169 floats]}` — equity vs one uniform-random villain hand |
| `_gen_summary.json` | machine-readable provenance of the generating run (seed, timing, benchmark results) |

## Hand order convention

The 169 canonical hands are emitted by walking the standard 13x13 grid
(rows/columns ordered `A K Q J T 9 8 7 6 5 4 3 2`) along its **upper triangle,
row by row**; the diagonal cell yields the pair, and each off-diagonal cell
yields the suited hand immediately followed by the offsuit hand:

```
for hi in [A, K, Q, ..., 2]:            # high rank
    emit hi+hi                          # pair, e.g. "AA"
    for lo in ranks below hi:           # low rank, descending
        emit hi+lo+"s"                  # suited, e.g. "AKs"
        emit hi+lo+"o"                  # offsuit, e.g. "AKo"
```

So the sequence is:

```
index   0..24 : AA AKs AKo AQs AQo AJs AJo ATs ATo A9s A9o ... A2s A2o
index  25..48 : KK KQs KQo KJs KJo ... K2s K2o
index  49..70 : QQ QJs QJo ... Q2s Q2o
...
index 165..168: 32s 32o 22
```

First hand is `AA` (index 0), last hand is `22` (index 168). Rank characters:
`A K Q J T 9 8 7 6 5 4 3 2` (ten is `T`). Both JSON files use this exact order.

## Method

* **Evaluator.** Fully vectorized numpy 7-card evaluator: per-suit 13-bit rank
  masks, bit-sliced rank counting (three bit-planes give per-rank counts 0-4
  with pure integer ops), and precomputed lookup tables over all 8192 13-bit
  masks for high-bit, straight detection (incl. the wheel A-5), and top-5-kicker
  packing. Hands map to a single comparable int64 (category * 13^5 + base-13
  tiebreaks). Throughput ~5.5M 7-card hands/s single-threaded.
  Validated against an independent brute-force best-of-C(7,5) reference
  evaluator: **0 ordering mismatches in 400,000 random hand pairs**
  (`python3 gen_equity169.py --selftest`), plus 0 mismatches in a separate
  700k-pair prototype validation including shared-board tie stress
  (fast/reference tie counts identical: 1567/1567 on 40k shared-board duels).
* **Matrix (`equity169.json`).** For each of the 14,196 unordered canonical
  matchups (upper triangle), equity is estimated by **stratified Monte Carlo**:
  every concrete suit-combo pair of the two hands consistent with card removal
  is a stratum with equal weight (this reproduces the exact uniform
  combo-averaging required), and 200,000 boards total per matchup are sampled
  uniformly from the remaining 48 cards (~200k/K per stratum for K strata).
  Ties count as half. The lower triangle is filled with the exact complement
  `equity[j][i] = 1 - equity[i][j]` **after rounding**, and the diagonal is
  defined as exactly `0.5` (the same-hand mirror equity, exact by symmetry).
  Values rounded to 4 decimals.
* **Vs-random (`rank_vs_random169.json`).** Derived **exactly** from the
  rounded matrix: for a fixed hero combo, a uniform-random villain hand is one
  of the C(50,2) = 1225 remaining combos; grouping by canonical hand gives
  card-removal weights `n_j` (identical across hero combos of the same hand by
  suit symmetry), so `vs_random[i] = sum_j n_j * equity[i][j] / 1225`.
  No extra sampling noise, and the two files are consistent by construction —
  `rank_vs_random169.json` is recomputable from `equity169.json` alone.
  Intended for ordering generic "top X%" ranges.

## Precision

* Per-entry Monte-Carlo standard error at N = 200,000 boards is at most
  `sqrt(0.25 / 200000) ≈ 0.0011` (less for lopsided matchups; the combo-mix
  variance component is eliminated by stratification).
* Agreement with exact exhaustive enumeration (all combo pairs x all C(48,5)
  boards) on spot-checked matchups is within ~1 SE — see the benchmark table.
* Complement identity holds **exactly** (max |eq[i][j] + eq[j][i] - 1| = 0.0),
  by construction. Diagonal is exactly 0.5.
* Vs-random values carry only the matrix noise averaged over ~168 entries
  (effective SE ~1e-4 plus 4-decimal rounding).

## Reproduction

```bash
cd assets/poker-resteal-lesson/tools
python3 gen_equity169.py --selftest          # evaluator self-test (~2 s)
python3 gen_equity169.py                     # full regeneration (~16 min)
```

Defaults: `--boards 200000`, `--seed 20260711`, output to `../data/`.
Requires python3 (3.9+) and numpy (built with 2.0.2); the output JSON is plain
data with no dependency on the environment used to generate it.

Generating run: matrix 912.9 s, vs-random derivation < 1 s
(macOS, single-threaded numpy).

## Known limits

* Matrix entries are Monte-Carlo estimates (SE ≤ ~0.0011), not exact
  enumeration; the acceptance spot-checks above bound the error empirically.
  Rounding is 4 decimals, so consumers should not read significance into the
  last digit beyond ±0.001.
* `equity[i][i] = 0.5` is a definition for the same-hand mirror (it is also the
  true value by symmetry when both players can hold the hand — ties dominate
  but wins/losses balance exactly).
* Card-removal effects **between** hero and villain are fully handled inside
  each matchup and in the vs-random weights; nothing here models multiway pots
  or dead cards from folded ranges.
